Method prototypes for the \doxylink{struct_rep_queue}{Rep\+Queue} class. 

A \doxylink{struct_rep_queue}{Rep\+Queue} is a queue of reports that are received from the Google Stadia Controller. The queue is used to store reports until they can be processed and sent out on UART.

typedef struct \mbox{\hyperlink{struct_stadia_rep}{Stadia\+Rep}} \mbox{\hyperlink{rep__queue_8h_a18a642d8c9a7dbb0ed7d84cce772d4cf}{Stadia\+Rep\+\_\+t}}}

STADIA HID REPORT STRUCTURE The HID report from the Stadia controller is a 10 byte sequence. The bytes are as follows\+:
\begin{DoxyItemize}
\item Byte 1\+: 4 bits of D-\/pad, 4 bits of padding
\item Byte 2\+: 8 bits of buttons
\item Byte 3\+: 7 bits of buttons, 1 bit of padding
\item Byte 4\+: 8 bits pointer X on stick
\item Byte 5\+: 8 bits pointer Y on stick
\item Byte 6\+: 8 bits pointer Z on stick
\item Byte 7\+: 8 bits pointer Rz on stick
\item Byte 8\+: 8 bits of Brake trigger
\item Byte 9\+: 8 bits of Throttle trigger
\item Byte 10\+: 3 bits for volume, play/pause, 5 bits of padding. (Used when a headset is connected directly to the controller. Not used in this project.) 
\end{DoxyItemize}
